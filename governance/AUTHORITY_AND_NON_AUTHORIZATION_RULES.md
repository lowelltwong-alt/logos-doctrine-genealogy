# Authority And Non-Authorization Rules

Machine-readable scaffold assertions:

```yaml
authority_level: interpretive_historical_profile_scoped
authorizes_doctrine_lineage_records: false
authorizes_source_imports: false
authorizes_source_rows: false
authorizes_reviewed_lineage_promotion: false
authorizes_graph_retrieval_vector_truth: false
authorizes_scripture_or_chunk_output: false
authorizes_new_relationship_verbs: false
authorizes_new_profiles: false
authorizes_new_enum_values: false
authorizes_theology_authority: false
authorizes_runtime_adapters: false
```

This repo exists so future doctrine-genealogy work can be staged safely. It does
not make the staged work authoritative.

Stop if a task attempts to treat AI output, semantic similarity, embeddings,
graph rank, generated confidence, issue text, comments, or PR bodies as review
or authority.
